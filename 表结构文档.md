# 数据库表结构文档

> AOI 复判系统 V4 — 基于 MySQL，字符集 `utf8mb4`，存储引擎 InnoDB

---

## 分表策略

每张业务表按 **热/冷数据** 分表，由 `TableRouter` 自动路由：

| 表后缀 | 范围 | 说明 |
|---|---|---|
| `_current` | 最近 30 天 | 热数据，频繁读写 |
| `_history_YYYYMM` | 按月归档 | 冷数据，按月分区，如 `_history_202601` |

路由逻辑：查询时间范围与 [今天-30天, 今天] 有交集 → 查 `_current`；涉及更早月份 → 依次查询对应 `_history_YYYYMM`。

## 表结构

### 1. `t_station_result` — 站点结果表（主表）

**每条记录 = 一个产品在一个站点的AOI检测结果 + 人工复判结果。**

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `Id` | INT | PK, AUTO_INCREMENT | 主键 |
| `User` | VARCHAR(100) | DEFAULT '' | 操作员 |
| `Line` | VARCHAR(50) | DEFAULT '' | 产线 |
| `WorkOrder` | VARCHAR(100) | DEFAULT '' | 工单号 |
| `MachineId` | VARCHAR(50) | DEFAULT '' | AOI设备ID |
| `Sn` | VARCHAR(200) | NOT NULL | 产品序列号 |
| `PackCode` | VARCHAR(200) | DEFAULT '' | 料号/包装码 |
| `ProductType` | VARCHAR(100) | DEFAULT '' | 产品型号 |
| `FixNo` | VARCHAR(50) | DEFAULT '' | 夹具号 |
| `HoleNo` | VARCHAR(50) | DEFAULT '' | 孔位号 |
| `FinalResult` | INT | DEFAULT 0 | AOI判定：0=待定, 1=OK, 2=NG |
| `ReviewResult` | INT | DEFAULT 0 | 复判结果：0=待定, 1=OK, 2=NG |
| `ReviewRemark` | VARCHAR(500) | DEFAULT '' | 复判备注 |
| `ReviewUser` | VARCHAR(50) | DEFAULT '' | 复判人 |
| `ReviewTime` | DATETIME | NULL | 复判时间 |
| `CreateTime` | DATETIME | NULL | 创建时间（AOI检测时间） |
| `UpdateTime` | DATETIME | NULL | 最后更新时间 |

**索引：**

| 索引名 | 字段 | 用途 |
|---|---|---|
| `idx_sn` | `Sn` | SN查询 |
| `idx_sn_ctime` | `Sn`, `CreateTime` | SN+时间排序 |
| `idx_final_ctime` | `FinalResult`, `CreateTime` | 良率统计按结果+时间 |
| `idx_workorder_ctime` | `WorkOrder`, `CreateTime` | 工单维度统计 |
| `idx_ctime` | `CreateTime` | 时间范围查询 |

---

### 2. `t_station_detail` — 站点明细表

**每条记录 = 产品经过的特定子站点的检测详情（含该站点的所有图像）。**

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `Id` | INT | PK, AUTO_INCREMENT | 主键 |
| `StationResultId` | INT | NOT NULL | 关联 `t_station_result.Id` |
| `Sn` | VARCHAR(200) | DEFAULT '' | 产品序列号 |
| `StationNo` | VARCHAR(20) | DEFAULT '' | 站点编号 |
| `StationName` | VARCHAR(100) | DEFAULT '' | 站点名称（如"上焊前检查"） |
| `StationType` | VARCHAR(50) | DEFAULT '' | 站点类型 |
| `StationResult` | INT | DEFAULT 0 | 该站点检测结果：0=待定, 1=OK, 2=NG |
| `StartTime` | DATETIME | NULL | 该站点开始检测时间 |
| `EndTime` | DATETIME | NULL | 该站点结束时间 |
| `AllImageUrls` | **JSON** | NULL | 该站点所有采集图像路径列表 |

**索引：**

| 索引名 | 字段 | 用途 |
|---|---|---|
| `idx_sn` | `Sn` | SN查询 |
| `idx_sn_station` | `Sn`, `StationNo` | SN+站点查询 |
| `idx_station_rid` | `StationResultId` | 关联主表查询 |
| `idx_start_time` | `StartTime` | 站点时间范围分析 |

---

### 3. `t_inspection_detail` — 检测明细表（核心分析表）

**每条记录 = 产品在某个站点的单个检测项（包含缺陷JSON数据），是质量看板分析的核心数据源。**

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `Id` | INT | PK, AUTO_INCREMENT | 主键 |
| `StationResultId` | INT | NOT NULL | 关联 `t_station_result.Id` |
| `StationDetailId` | INT | NOT NULL | 关联 `t_station_detail.Id` |
| `Sn` | VARCHAR(200) | DEFAULT '' | 产品序列号 |
| `StationNo` | VARCHAR(20) | DEFAULT '' | 站点编号 |
| `Result` | INT | DEFAULT 0 | 检测结果：0=待定, 1=OK, 2=NG |
| `SingleImagePath` | VARCHAR(500) | DEFAULT '' | 缺陷裁剪图路径 |
| `ImageWidth` | INT | DEFAULT 0 | 图像宽度 |
| `ImageHeight` | INT | DEFAULT 0 | 图像高度 |
| `Measurements` | **JSON** | NULL | 测量项数据（含嵌套缺陷列表） |
| `Defects` | **JSON** | NULL | 缺陷列表数据 |
| `Time` | DATETIME | NULL | 检测时间 |

**索引：**

| 索引名 | 字段 | 用途 |
|---|---|---|
| `idx_sn` | `Sn` | SN查询 |
| `idx_sn_station` | `Sn`, `StationNo` | SN+站点 |
| `idx_rid_did` | `StationResultId`, `StationDetailId` | 关联查询 |
| `idx_result_time` | `Result`, `Time` | 按结果分类+时间统计 |
| `idx_time` | `Time` | 时间范围查询 |

## JSON 字段结构详解

`t_inspection_detail.Measurements` 和 `t_inspection_detail.Defects` 存储嵌套 JSON，结构如下：

### Measurements / Defects JSON Schema

```json
{
  "Datas": [
    {
      "DefectName": "缺件",              // 缺陷名称（也兼容 "defectName"）
      "DefectType": "电气",              // 缺陷类型
      "Level": "Critical",              // 缺陷等级
      "Count": 1,                       // 同类缺陷数量
      "AreaSize": 12.5,                 // 缺陷面积 (px²)
      "ImageWidth": 640,                // 子图宽度
      "ImageHeight": 480,               // 子图高度
      "CenterPoint": {                  // 缺陷中心坐标
        "X": 320,
        "Y": 240
      },
      "BoundingRect": {                 // 缺陷外接矩形
        "X": 300,
        "Y": 220,
        "Width": 40,
        "Height": 40
      },
      "ContourPoints": [                // 缺陷轮廓点集
        {"X": 300, "Y": 220},
        {"X": 340, "Y": 220},
        {"X": 340, "Y": 260},
        {"X": 300, "Y": 260}
      ],
      "NineGridArea": "5",              // 九宫格区域 (1-9)
      "ProductArea": "A区",             // 产品功能区域（用于热力图）
      "AlgorithmVersion": "v2.1.3",     // 检测算法版本
      "DefectImagePath": "D:/images/defect_001.jpg"  // 缺陷子图路径
    }
  ]
}
```

> **兼容性说明**：字段名 `DefectName` / `ProductArea` 同时兼容全小写 `defectName` / `productArea` 格式。

## 表间关系（ER）

```
t_station_result (1)
    Id (PK)
    │
    ├── StationResultId ──→ t_station_detail (N)
    │                           Id (PK)
    │                           StationResultId (FK)
    │
    └── StationResultId ──→ t_inspection_detail (N)
                                Id (PK)
                                StationResultId (FK)
                                StationDetailId (FK → t_station_detail.Id)
```

- **`t_station_result` 1 : N `t_station_detail`** — 一个产品经过多个子站点
- **`t_station_result` 1 : N `t_inspection_detail`** — 一个产品有多个检测项
- **`t_station_detail` 1 : N `t_inspection_detail`** — 一个子站点下多个检测项

## 分析查询示例

### 良率趋势（按天聚合）
```sql
SELECT DATE_FORMAT(CreateTime, '%Y-%m-%d') AS period,
       COUNT(*) AS total,
       SUM(CASE WHEN FinalResult = 1 THEN 1 ELSE 0 END) AS ok,
       SUM(CASE WHEN FinalResult = 2 THEN 1 ELSE 0 END) AS ng
FROM t_station_result_current
WHERE CreateTime BETWEEN '2026-01-01' AND '2026-01-31'
GROUP BY period ORDER BY period;
```

### 缺陷分布（解析 JSON）
```sql
SELECT Measurements, Defects
FROM t_inspection_detail_current
WHERE Time BETWEEN '2026-01-01' AND '2026-01-31';
```
→ 应用层解析 JSON 中 `Datas[].DefectName` 字段，Counter 计数得到 Top-N 缺陷分布。

### 站点 NG 率
```sql
SELECT StationName, StationResult, COUNT(*) AS cnt
FROM t_station_detail_current
WHERE StartTime BETWEEN '2026-01-01' AND '2026-01-31'
GROUP BY StationName, StationResult;
```

## 附：数据流全景

```
AOI 设备检测
    │
    ▼
t_station_result (FinalResult=OK/NG)
    │
    ├── t_station_detail (各子站点结果 + AllImageUrls)
    │
    └── t_inspection_detail (每个检测项 + Measurements/Defects JSON)
                │
                ├── 质量看板：YieldAnalyzer → 良率趋势
                ├── 质量看板：DefectAnalyzer → 缺陷分布 / Pareto / 记录表
                ├── 质量看板：HeatmapAnalyzer → 区域热力图
                │
                └── 复判操作：读取缺陷图 → 人工判定 → 更新 ReviewResult
```
